- PERF-01: `scan_code_health` rebuilds the inventory (walk + read + SHA-256) ≥8×

- PERF-02: `rglob("*")` descends into `.git`/`node_modules`/`.venv` before
  filtering (pathlib has no prune hook) — walk is O(all files on disk).
- PERF-03: search tools re-walk + hash the whole repo per call for a path list
  already in the `files` table.
  → Wants a scan-determinism characterization test (TEST-03) landed first.

## Findings considered and rejected (do not re-audit)

Verified as non-issues by the investigators or during vetting:

- **Raw `sqlite3` in the transport layer** — already removed; no `sqlite3`
  import remains in `mcp/`, `cli/`, or `dashboard/` (a prior audit's fix landed).
- **`_record_suppressions` writing `stable_key` into the `finding_events.finding_id`
  FK column** — safe: `build_finding` sets `id == stable_key` (`engine/rules/model.py:73`),
  so the FK resolves; no crash.
- **`answer_pack` token-budget math** — verified correct: terminates and reports
  honestly (a recent commit fixed the query-exceeds-budget case).
- **Division-by-zero at `search_run.py:269`, `precision.py:242`,
  `refactor_planning.py:298`** — all guarded.
- **`defusedxml` unused** — it *is* used (`services/coverage.py:7-8`); XML
  parsing is safe.
- **Dead modules** — a collision-aware zero-import scan of all 133 non-`__init__`
  modules found none unreferenced.
- **Multiple repo-root resolvers** — single `core/paths.resolve_repo_root` used
  everywhere.
- **`test_frecency_ranking` flakiness** — not flaky; it injects `now=`/`timedelta`
  to avoid real sleeps.
- **Read-path traversal / symlink escape** — containment is real and
  symlink-safe (`normalize_repo_path`, `core/paths.py:19`); the inventory walk
  does not descend escaping symlinks.
- **`codescent ci` missing SARIF / baseline / PR output** — already ships all of
  those; not a gap.
- **Dashboard export-without-import asymmetry** — by design (read-only local
  dashboard is an explicit non-goal).
- **`typer`/`pydantic` version lag, Python 3.12 floor** — current and non-EOL;
  no cost to the lag. Not worth doing.
